# Doubles Scheduler — Agent Instructions

## プロジェクト

ラケットスポーツ向けソーシャルダブルスの次ラウンド組み合わせを生成する TypeScript ライブラリ（`doubles-scheduler`）。

詳細仕様: [`docs/SPEC.md`](docs/SPEC.md)

## 仕様ドキュメントの同期（必須）

仕様の追加・変更がある場合、**コード変更と同じ作業単位で** `docs/SPEC.md` と `README.md` を更新する。

| ファイル | 役割 |
| --- | --- |
| `docs/SPEC.md` | 正本（API、型、ルール、設計判断、テスト観点） |
| `README.md` | 概要・クイックスタート・利用者向け要約 |

### 更新トリガー

- 公開 API / 型の追加・変更
- データモデル・状態構造の変更
- ビジネスルール・制約・アルゴリズムの変更
- 未決事項（SPEC §18.2）の解消
- フェーズ計画・スコープの変更

### 手順

1. `docs/SPEC.md` を先に更新
2. `README.md` の要約を同期（詳細は SPEC へ委譲）
3. SPEC 末尾「改訂履歴」に追記
4. 実装とドキュメントの矛盾がないことを確認

### 禁止

- 仕様未更新のまま挙動だけ変更しない
- 詳細を README のみに書いて SPEC を放置しない

## 実装前

`docs/SPEC.md` を読んでから作業を開始する。
